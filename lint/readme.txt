Порядок создания списка особенностей:
1. Заполняем и выгружаем настройки конфигурации. Исключаем подсистему СтандартныеПодсистемы
2. Когда собираем данные и проверяем код. Тогда получаем все ошибки конфигурации
3. Убеждаемся, что ЖР нет ошибок соединения с яндекс словарем. Если есть, то нужно подложить платформе файл с настройками прокси
4. Загружаем файл особенностей используемой версии БСП
5. Когда Выполняем проверку всех объектов по собранным данным И включаем регистрацию всех ошибок как особенностей. Тогда получаем 0 ошибок конфигурации и много особенностей
6. Если на предыдущем шаге получили несколько ошибок, тогда можно эти ошибки через отчет перевести в особенности по причине "Технический долг" или "Проектное решение БСП"

Изменения для АПК 1.2.5.16
1. в шаблон ПравилаОбменаДанными обработки УниверсальныйОбменДаннымиXML в правило для Перечисления Типы объектов добавлено
                    <Значение>
                        <Код>62</Код>
                        <Наименование>HTTP-сервис --&gt; HTTP-сервис</Наименование>
                        <Порядок>3250</Порядок>
                        <Источник>HTTPСервис</Источник>
                        <Приемник>HTTPСервис</Приемник>
                    </Значение>
2. В файле настроек конфигурации checkParams.xml русские значения "Да" заменены на "true" или "yes".Например,
           <СтрокаТабличнойЧасти>
                <Использовать>true</Использовать>
3. В файле настроек конфигурации checkParams.xml пустая дата время заменены пустой строкой. Например,
Было:
          <ВремяВыгрузкиОсобенностей>01.01.0001 0:00:00</ВремяВыгрузкиОсобенностей>
Стало:
          <ВремяВыгрузкиОсобенностей></ВремяВыгрузкиОсобенностей>
